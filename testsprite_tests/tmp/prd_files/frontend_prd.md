# Frontend Product Requirements Document (PRD) – TestSprite Configuration

## 1. Overview
This document defines the testing configuration for the **AgriSentinel AI** frontend application. It is intended for use with **TestSprite** to generate automated UI tests that verify the end‑to‑end user experience, visual components, and data flows.

## 2. Testing Types
- **Functional UI Tests** – Verify that each interactive component (navbar, decision cards, analytics charts, voice synthesis button, etc.) behaves as expected.
- **Visual Regression** – Capture screenshots of key screens and compare against baselines to detect unintended UI changes.
- **Performance Checks** – Measure page load time, Time‑to‑Interactive (TTI), and asset size.
- **Accessibility** – Ensure WCAG‑AA compliance (ARIA labels, focus order, color contrast).

## 3. Mode
- **Frontend** – Tests run against the compiled Next.js client bundle.

## 4. Scope
- **Codebase** – The entire codebase is under test, covering all pages (`/dashboard`, `/analytics`, `/`, etc.).

## 7. Product Specification Document
> **Upload a detailed product spec** (PDF/Markdown) that describes the desired features, user flows, and acceptance criteria. This file will be referenced by TestSprite when generating test cases.

### Example Specification Outline
1. **User Journey** – Image upload → Disease detection → Decision score → Profit simulation → Alerts → Mandi recommendation.
2. **Key UI Components** – Navbar, DecisionEngine, ProfitSimulationCard, AlertsCard, BestMandiCard, AnalyticsPage.
3. **Success Criteria** – All components render within 2 seconds, voice synthesis button reads the recommendation, offline mode works after first load.
